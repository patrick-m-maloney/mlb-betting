  1. What ProjectionEngine currently does (Layer 0) and how good/bad it is                                                                     
                                                                                                                                               
  What it does: projections.py:68-103 calculates a single career PA-weighted wOBA per player across all game-by-game logs (2017-2025). It joins
   each game's counting stats (BB, HBP, 1B, 2B, 3B, HR) with that season's linear weights, sums the weighted numerator, divides by total career
   PA. Each player gets one number that never changes game-to-game.                                                                          
                                                                                                                                               
  How bad this is as a baseline:                                                                                                               
  
  - No recency weighting. A guy who hit .340 wOBA from 2017-2022 and then fell off a cliff in 2023-2025 still gets credit for those peak years.
   Career averages are dragged around by irrelevant seasons.                                                                                 
  - No aging curve. A 36-year-old with a .350 career wOBA and a 23-year-old rookie with a .290 career wOBA — the model has no idea the old     
  player is likely declining or the rookie might be improving.                                                                                 
  - No platoon/park/matchup effects. Every player gets the same projection regardless of who's pitching or where the game is.
  - Massive fallback rate. You have 2,309 batters in the game logs, but lineups contain bbref_ids that need to map through                     
  player_id_map.parquet. Any player who fails the crosswalk silently gets league_avg_woba (~0.315). So for some games, multiple lineup spots   
  are just league average.                                                                                                                     
  - No pitching input. The Monte Carlo sim only projects runs via batting lineup wOBA. The opposing starter's quality is completely ignored.   
                                                                                                                                               
  That said, it's a reasonable Layer 0. Career wOBA is still informative — it separates Juan Soto from a replacement-level hitter. The bigger  
  issue is that the spread between teams is compressed. When everyone regresses toward their career mean, the model will produce win           
  probabilities clustered around 45-55%, which means it rarely finds large edges against the market. Markets are already pricing in most of    
  that career-level information.

  After KNN comps integration, you'd expect:                                                                                                   
  - Better handling of early-career players (comps fill in what career avg can't)
  - Recency-sensitive projections (the Kalman blend step weights recent actuals)                                                               
  - Wider team-level spreads on any given day — and therefore more/larger edge signals
  - Practically: probably a 3-8% shift in win probability for games where key players are on breakout/decline trajectories that career avg     
  misses

  4. Full data flow with KNN integrated                                                                                                        
  
  Step-by-step from lineup arrival to win probability:                                                                                         
                  
  1. Lineup arrives — lineups.py scrapes Rotowire, saves to bronze/silver parquet with bbref_ids per batting order slot                        
  2. ProjectionEngine.get_player_woba(bbref_id) — for each of the 9 batters in each lineup:
    - Map bbref_id → mlbID via player_id_map.parquet                                                                                           
    - Map mlbID → FanGraphs IDfg (need to add this crosswalk)                                                                                  
    - Layer 1 (KNN): Pull this player's current-season stats from FG leaderboards. Feed age, service_time, PA, wRC+, ISO, K%, BB%, BABIP,      
  HardHit%, Barrel%, Spd into the fitted KNN model. Get k=15 nearest historical comps. Weighted-average their next-season (or rest-of-season)  
  stat deltas → KNN projected wOBA                                                                                                             
    - Layer 2 (Kalman blend): If player has N games this season, blend: woba = (1 - confidence(N)) * knn_projection + confidence(N) *          
  actual_2026_woba. Confidence grows with PA (e.g., confidence = min(PA / 400, 0.8))                                                           
    - Return the blended wOBA
  3. MonteCarloSimulator.project_lineup_runs() — multiply each batter's blended wOBA by their batting-order PA weight, sum, scale by           
  woba_to_runs constant and home/away multiplier → projected mean runs for that team                                                           
  4. MonteCarloSimulator.simulate_game() — draw 10,000 samples from Normal(projected_runs, 2.6) for each team, clipped at 0. Count away_wins vs
   home_wins → win probability                                                                                                                 

  5. EdgeDetector.find_edges() — compare model win_prob to implied probabilities from The Odds API, Kalshi, Polymarket. Flag where |model - 
  market| > threshold
